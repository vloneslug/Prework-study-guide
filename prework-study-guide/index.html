HTML

  • !DOCTYPE tells the interface/other developers what type of language is being used
  • head element contain information about the webpage
  • meta element contains metadata that is not seen by the user but helps browser define certain features for certain users
  • body element represents VISIBLE content shown to the user
  • src ATTRIBUTE defines location or source of an image file
  • alt ATTRIBUTE contains text string that decribes appearance/funcionality of image(can help visually impaired users)
  • table element creates a table
  • tr element adds ROWS to a table
  • td adds DATA to a table
  • th creates table HEADINGS that clarify meaning of the data
  • colspan allows data to span over multiple COLUMNS
  • rowspan allows data to span over multiple ROWS
  • tables can be split into three main sections: HEAD, BODY, & footer
  • thead creates a TABLE HEAD
  • tbody creates a TABLE BODY
  • tfoot creates a table FOOTER
  • section element helps me to group data together(its how im able to section off each group of notes!)

GIT

  • Git is a free and open source VERSION CONTROL system designed to handle projects with speed and efficiency
  • REPO/REPOSITORY - a folder thats holds all files and revision history for projects
  • ISSUE - Open/create issues to report NEW TASKS and close once task is complete: bug fixes, new features, updates
  • BRANCH - workspace where i can make changes that DONT affect the LIVE SITE: create branch and name it to best describe action being taken
  • COMMIT - Adding, editing, or deleting a file is making a commit... Commits wills add changes to the new branch and save new version
  • PULL REQUEST(PR) - asks for changes that have been made to be pulled/merged into another branch: Also a good way to get a review of changes by self or others
  • MERGE - when a PR is approved changes are pulled in/MERGED... these changes will be published/deployed onto the live site
  • MARKDOWN(.md) - the way content files are formatted in github... a way to write for the web using plain text that easily convert to html
  • GitHub workflow is the Creation of Branch.. Commits added.. Open Pull Request.. Discuss and review.. Merge and Deploy
  • Terminal/VSCODE Steps for Git..
    1. cd until in proper directory
    2. code . to open vscode
    3. git status to check git status
    4. git switch or checkout to move the current Branch
    5. git checkout -b to create new branch if necessary
    6. git checkout -b feature/name-of-new-branch to name the new branch
    7. check git status
    8. git add -A adds all modifications into the current branch STAGING AREA
    9. git commit -m "description" commits modifications and describes what the commit contains
    10. git pull pulls modifications into the branch.. always do git pull to ensure youre working on the most updated version
    11. git push pushes local changes to the remote github branch
